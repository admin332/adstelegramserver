

## Цель
Исправить отображение рейтинга для каналов без отзывов: показывать **"0 (нет отзывов)"** вместо **"4.5 (нет отзывов)"**.

## Проблема

Сейчас в базе данных триггер `update_channel_rating()` устанавливает дефолтный рейтинг 4.5 когда нет отзывов:
```sql
SELECT COALESCE(ROUND(AVG(rating)::numeric, 1), 4.5)
```

И в UI этот рейтинг отображается даже когда `reviewsCount = 0`.

## Решение

Достаточно исправить только UI логику — показывать "0" когда нет отзывов, независимо от значения в базе.

### Изменение в `src/pages/Channel.tsx`

**Было (строка 97):**
```tsx
return `${channel.rating} (нет отзывов)`;
```

**Станет:**
```tsx
return `0 (нет отзывов)`;
```

Или для большей ясности, можно переписать функцию `getRatingValue()`:

```tsx
const getRatingValue = () => {
  const count = channel.reviewsCount || 0;
  if (count > 0) {
    return `${channel.rating} (${count} ${pluralize(count, 'отзыв', 'отзыва', 'отзывов')})`;
  }
  return '0 (нет отзывов)';
};
```

## Результат

До:
```
⭐ Рейтинг          4.5 (нет отзывов)
```

После:
```
⭐ Рейтинг          0 (нет отзывов)
```

## Примечание

Дефолтный рейтинг 4.5 в базе данных можно оставить — он будет использоваться только как fallback и не будет отображаться пользователям, когда отзывов нет.
