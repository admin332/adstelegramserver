

# Замена кнопки на ссылку продукта для prompt-кампаний

## Суть изменения

Для prompt-кампаний вместо кнопки используем **ссылку на продукт**, которую владелец канала сам вставит в текст поста. Это сохранит форматирование при пересылке через `copyMessage`.

## Изменения в форме создания кампании

### Шаг 3: Переименовать "Референсы" → "Ссылка на продукт"

**Было:**
- Заголовок: "Референсы"
- Описание: "Добавьте ссылки на примеры или материалы для автора (опционально)"
- Подсказка: "Примеры постов, ваш сайт..."

**Станет:**
- Заголовок: "Ссылка на продукт"
- Описание: "Добавьте ссылку, которую автор вставит в пост"
- Подсказка: "Автор добавит эту ссылку в текст поста"
- Поле остаётся опциональным

### Шаг 4: Убрать блок кнопки для prompt-кампаний

Для `isPromptMode === true` скрыть блок:
```tsx
{/* Показывать кнопку только для ready_post */}
{!isPromptMode && (
  <div className="space-y-2">
    <Label>Кнопка (опционально)</Label>
    ...
  </div>
)}
```

## Файл к изменению

| Файл | Изменение |
|------|-----------|
| `src/components/create/CreateCampaignForm.tsx` | Переименовать шаг 3 для prompt, скрыть кнопку на шаге 4 для prompt |

## Результат

### Prompt-кампания (Native Ads):
- **Шаг 3**: "Ссылка на продукт" — ссылка, которую автор вставит в текст
- **Шаг 4**: Только бриф, без полей кнопки

### Ready_post кампания:
- **Шаг 3**: Загрузка медиа (без изменений)
- **Шаг 4**: Текст + кнопка (без изменений)

## Флоу для prompt-кампании

```
1. Рекламодатель вводит ссылку на продукт (шаг 3)
2. Рекламодатель пишет бриф (шаг 4)
3. Владелец канала получает бриф + ссылку
4. Владелец пишет пост и сам вставляет ссылку в текст
5. Пост пересылается рекламодателю через copyMessage (форматирование сохраняется)
```
